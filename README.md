 # Longread: анкеты студентов
 
 Учебный longread‑сайт с анкетами студентов. Проект сделан в рамках задания по командной разработке с использованием Git и GitHub.
 
 ## Команда
 
 - **Ионов Роман Андреевич** — Team Lead, frontend. Отвечает за структуру проекта, ветки, Pull Request'ы и общие стили страницы.
 - **Феталиев Артем Эдуардович** — Frontend разработчик. Отвечает за свою секцию с анкетой и проработку деталей интерфейса.
 - **Бобылев Антон Дмитриевич** — Frontend разработчик. Отвечает за свою секцию с анкетой и визуальное наполнение.
 
 ## Технологии
 
 - Чистый **HTML**
 - Чистый **CSS**
 - Без JavaScript и сложных анимаций
 
 ## Структура проекта
 
 - `index.html` — основная страница longread с тремя анкетами.
 - `styles.css` — общие стили для всей страницы.
 - `img/` — фотографии студентов (аватары для анкет).
 - `.gitignore` — исключения для Git.
 - `CHANGELOG.md` — история изменений проекта.
 - `README.md` — описание проекта и команды.
 
 ## Ветки
 
 Принята следующая схема веток:
 
 - `main` — основная стабильная ветка (релизная).
 - `develop` — рабочая ветка, в которую вливаются feature‑ветки.
 - `feature/*` — ветки для разработки отдельных задач:
   - `feature/ionov-section` — секция Романа.
   - `feature/fetaliev-section` — секция Артёма.
   - `feature/bobylev-section` — секция Антона.
 - `hotfix/*` — срочные исправления из `main`.
 
 ## Рекомендованный процесс работы
 
 1. Тимлид создаёт ветку `develop` от `main` и пушит её на GitHub.
 2. Каждый участник создаёт свою ветку от `develop`:
 
    - `feature/ionov-section`
    - `feature/fetaliev-section`
    - `feature/bobylev-section`
 
 3. Вносит изменения только в своей ветке и делает осмысленные коммиты:
 
    - `feat:` — новая функциональность (новые блоки, секции).
    - `fix:` — исправления (ошибки, опечатки, верстка).
    - `style:` — правки по стилю (отступы, шрифты, цвета).
    - `refactor:` — рефакторинг без изменения логики.
    - `docs:` — документация (`README`, `CHANGELOG` и т.д.).
 
 4. После завершения задачи открывает Pull Request из своей `feature/*` ветки в `develop`.
 5. Тимлид делает ревью, при необходимости просит правки, затем мержит PR.
 6. Когда все секции готовы и протестированы, тимлид сливает `develop` в `main`.
 7. При обнаружении критической ошибки на прод‑версии создаётся ветка `hotfix/*` от `main`, вносится минимальное исправление и через PR мержится обратно в `main` (и затем в `develop`).
 
 ## Как запустить
 
 1. Клонировать репозиторий:
 
    ```bash
    git clone https://github.com/Radik655/frontend-longread-team-lead.git
    cd frontend-longread-team-lead
    ```
 
 2. Открыть файл `index.html` в браузере (двойной клик или через «Открыть с помощью…»).
